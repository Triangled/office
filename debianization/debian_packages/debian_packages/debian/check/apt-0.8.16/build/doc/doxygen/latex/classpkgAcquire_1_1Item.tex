\subsection{\-Detailed \-Description}
\-Represents the process by which a \doxyref{pkg\-Acquire}{p.}{classpkgAcquire} object should \{\{\{ retrieve a file or a collection of files. 

\-By convention, \doxyref{\-Item}{p.}{classpkgAcquire_1_1Item} subclasses should insert themselves into the acquire queue when they are created by calling \doxyref{\-Queue\-U\-R\-I()}{p.}{classpkgAcquire_1_1Item_ab455ec87a776308e2892d3a383b1c2d2}, and remove themselves by calling \doxyref{\-Dequeue()}{p.}{classpkgAcquire_1_1Item_a717924cd0aecc1908058a6ed7ffd31f3} when either \doxyref{\-Done()}{p.}{classpkgAcquire_1_1Item_a688635545a65b9a4bebed86627574d41} or \doxyref{\-Failed()}{p.}{classpkgAcquire_1_1Item_ad7c33b8520f000b8dfdf48431ddf2949} is invoked. \doxyref{\-Item}{p.}{classpkgAcquire_1_1Item} objects are also responsible for notifying the download progress indicator (accessible via \doxyref{\-Owner}{p.}{classpkgAcquire_1_1Item_a76f8c48f1e7f5669a16a9d5e863edd10}-\/$>$\-Log) of their status.
