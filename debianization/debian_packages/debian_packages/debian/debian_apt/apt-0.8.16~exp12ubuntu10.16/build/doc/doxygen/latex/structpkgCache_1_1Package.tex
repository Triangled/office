\subsection{\-Detailed \-Description}
contains information for a single unique package 

\-There can be any number of versions of a given package. \doxyref{\-Package}{p.}{structpkgCache_1_1Package} exists in a singly linked list of package records starting at the hash index of the name in the \doxyref{pkg\-Cache\-::\-Header\-::\-Pkg\-Hash\-Table}{p.}{structpkgCache_1_1Header_a32cd30c95a099e40e2199ecb1036d744}

\-A package can be created for every architecture so package names are not unique, but it is garanteed that packages with the same name are sequencel ordered in the list. \-Packages with the same name can be accessed with the \doxyref{\-Group}{p.}{structpkgCache_1_1Group}.

\-Base of a singly linked list of versions. 

\-Each structure represents a unique version of the package. \-The version structures contain links into \doxyref{\-Package\-File}{p.}{structpkgCache_1_1PackageFile} and the original text file as well as detailed information about the size and dependencies of the specific package. \-In this way multiple versions of a package can be cleanly handled by the system. \-Furthermore, this linked list is guaranteed to be sorted from \-Highest version to lowest version with no duplicate entries.
